# 프로젝트명: GG-SHOP

## 1. 프로젝트 소개
본 프로젝트는 개인 학습을 목적으로 한 상품 판매 웹 서비스입니다.  
회원은 상품을 조회하고 주문할 수 있으며, 관리자는 상품 및 주문을 관리할 수 있습니다.

## 2. 기획 의도
- 실제 이커머스 서비스 구조를 이해하기 위해 기획부터 구현까지 단독으로 진행
- 요구사항 정의 &rarr; 설계 &rarr; 구현 순서를 따르는 것을 목표로 함

## 3. 주요 기능
### 회원
- 회원가입 / 로그인
- 주문 내역 조회

### 상품
- 상품 목록 조회
- 상품 상세 조회

### 주문
- 장바구니 기능
- 주문 및 결제

### 관리자
- 상품 등록 / 수정 / 삭제
- 주문 상태 관리

## 4. 요구사항 정의

### 회원 사용자
- 회원은 회원가입을 할 수 있다
- 회원은 로그인을 할 수 있다
- 회원은 상품 목록 및 상세 정보를 조회할 수 있다
- 회원은 상품을 장바구니에 담을 수 있다
- 회원은 주문을 생성할 수 있다
- 회원은 자신의 주문 내역을 조회할 수 있다

### 관리자
- 관리자는 상품을 등록, 수정, 삭제할 수 있다
- 관리자는 주문 상태를 변경할 수 있다

### 비기능 요구사항
- 비밀번호는 암호화하여 저장한다
- 인증이 필요한 기능은 로그인 후 접근 가능하다

## 5. 화면 설계
- 로그인 / 회원가입 페이지
- 상품 목록 페이지
- 상품 상세 페이지
- 장바구니 페이지
- 주문 / 결제 페이지
- 관리자 상품 관리 페이지

## 6. 데이터베이스 설계
### User
- id
- email
- password
- role

### Product
- id
- name
- price
- stock

### Order
- id
- user_id
- status
- total_price

### OrderItem
- id
- order_id
- product_id
- quantity
- price
> 하나의 주문은 여러 개의 주문 상품(OrderItem)을 가진다

## 7. API 설계

| Method | Endpoint | Description | Auth |
|------|--------|-----------|------|
| GET | /products | 상품 목록 조회 | ❌ |
| GET | /products/{id} | 상품 상세 조회 | ❌ |
| POST | /orders | 주문 생성 | ✅ |
| GET | /orders/my | 내 주문 조회 | ✅ |
| POST | /admin/products | 상품 등록 | 관리자 |

## 8. 기술 스택
- Frontend: React
- Backend: Spring Boot
- Database: PostgreSQL
- Auth: JWT
